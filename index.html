Scienza

Studio nutrizionale

Calorie e metabolismo

La app calcola il metabolismo basale (MB) e il fabbisogno energetico in base a età, peso, altezza e genere, poi propone deficit, mantenimento o surplus.

Risultato finale

Vengono stimate le calorie nette, cioè l’energia assunta meno quella bruciata, per comprendere meglio il ciclo: mangiare, muoversi, bilanciare.

Fisica

Analisi del movimento e dell’energia: come l’attività fisica trasforma il corpo e quantifica il dispendio energetico di camminate, sport e altre azioni.

Energia in movimento

Laboratorio e cyclette

Nella prima parte abbiamo analizzato il legame tra alimentazione e dispendio energetico. Usando la cyclette, abbiamo simulato il lavoro meccanico richiesto per “bruciare” uno snack.

Formule chiave: W = F·s, v = s/t, kcal consumate = MET × peso × ore, 1 kcal = 4186 J.

Test di potenza sulle scale

Nella seconda fase la classe ha provato la potenza esplosiva sui corridoi salendo le scale alla massima velocità, mentre i compagni cronometravano la salita.

Potenza: W = m·g·h e P = W/t, con m massa, g accelerazione di gravità (9.81 m/s²), h dislivello e t tempo impiegato.

Conclusioni fisiche

L’esperimento ha mostrato la differenza tra energia totale e potenza: salire le scale velocemente richiede molta potenza, ma l’energia dipende soprattutto da peso e altezza delle scale.

Dettagli Nutri-track

L’app nasce per aiutare gli studenti a valutare il proprio fabbisogno calorico e capire come integrare alimentazione e movimento.

Funzionalità principali

  • Calorie ingerite

    Inserisci alimenti e quantità per stimare l’apporto giornaliero.

  • Calorie bruciate

    Registra le attività fisiche anche leggere, come una passeggiata di 20 minuti.

  • Fabbisogno

    Calcola il metabolismo basale maschile o femminile e propone deficit, mantenimento o surplus.

Il gruppo ha utilizzato GitHub per il codice, Netlify per il deployment, Supabase per gestire i dati utente e le API di Google Sheets per esportare i dati.

Parte tecnica

Spiegazione chiara del funzionamento front-end e back-end dell’app, con i servizi e gli strumenti usati per costruire il progetto.

Architettura web

Front-end

La pagina è costruita con HTML per i contenuti, CSS per il look e il layout, e JavaScript per i calcoli, i grafici e l’interazione dell’utente.

HTML

Struttura delle sezioni, titoli, form e risultati.

CSS

Palette fissa, spaziature, bordi arrotondati e design responsive.

JavaScript

Calcolo del metabolismo, indicatori dinamici e aggiornamento dei risultati in tempo reale.

Back-end

Per una app reale, il back-end gestisce i dati, l’autenticazione e l’archiviazione: GitHub per il codice, Netlify per il deploy, Supabase per il database e Google Cloud per i fogli.

GitHub

Salvataggio del progetto, versionamento e collaborazione del gruppo.

Netlify

Pubblicazione automatica del sito statico e hosting sicuro.

Supabase

Database serverless per salvare utenti, preferenze e dati di consumo.

Come si collega

La versione presente è una web app statica: il front-end esegue i calcoli nel browser. In un progetto completo, il back-end cloud fornisce API e salvataggio persistente.

Esempio architettura: GitHub → Netlify (deploy) + Supabase (database) + Google Cloud Sheets (esportazione dati) + JavaScript lato client.

Guida: come creare una web app

Passi chiari per costruire una web app, perfetti per il lavoro di gruppo e per spiegare l’intero processo alla classe.

1. Scegli l’idea

Definisci lo scopo: nel nostro caso l’app deve aiutare a gestire calorie, attività e suggerire piani alimentari.

2. Progetta la struttura

Organizza le sezioni del sito: introduzione, descrizione, demo, guida e team. Usa una palette fissa e un layout semplice.

3. Costruisci il design

Usa HTML per i contenuti, CSS per lo stile e JavaScript per l’interattività. Evita colori troppo sgargianti.

4. Aggiungi interattività

Implementa calcoli in tempo reale, tab con informazioni e grafici semplici per rendere il sito vivo.

5. Testa in gruppo

Prova la web app con compagni e professori, correggi bug e migliora la chiarezza dei testi.

6. Pubblica il sito

Usa Netlify o GitHub Pages per pubblicare. Collega il progetto a GitHub per gestire versioni e aggiornamenti.

Vibes coding: costruisci l’app con passi semplici, inizia dal contenuto, poi dal layout, quindi aggiungi le funzioni e infine pubblica.
